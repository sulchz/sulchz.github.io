Background

I'd been interested in tech and software for a while, and had some fun in school doing stuff with AutoHotKey (Windows only). I got into Linux late 2018, on Manjaro, largely due to wanting to customise stuff a lot more than Windows would let me. At some point I'll be adding pictures of stuff I made, some of which I'm quite proud of :p. I started off on i3wm, which I still recommend to people new to tiling window managers simply due to how popular it is and the on-the-fly recompiling, it's a lot simpler to use than stuff like DWM which is what I'm on now. Again, at some point I need to point to the current configs which I use. Feel free to use them/edit them if you'd like.

Anyways, after Manjaro I spent a lot of time bouncing around various distros; I had several spare PCs and no shortage of time. My recommendations now are (in no particular order):

  • Arch Linux - If you're willing to tinker around with stuff it's great simply due to how good the wiki is. Also, pacman is one of the better package managers I've used; the AUR is amazing as well (and probably the only thing keeping me here haha).
  • Void Linux - Another really nice bleeding-edge distro, with a very nice package manager. There were some schenanigans regarding the bus-factor with it a while back but it seems to have been resolved. It uses musl and runit as well, vs glibc and systemd, if that makes a difference.
  • Debian (netinst) - Surprisingly 'quiet' option. Really small install if you deselect everything, and afterwards you have a nice little system. I don't really see the point of running some downstream variant, particularly when you can build as light of a system from scratch, and will probably have a rant about this at some point (!!!).
I have also spent a fair bit of time exploring some more obscure stuff. Here are some nice projects worth looking at:
  • Plan 9 - A cool little project from Bell Labs, or rather set of projects as there are several forks. Not to be really used as a daily driver (unless you're into that, I suppose) but worth looking at for the novelty.
  • KISS Linux - A small project but one that I really like in terms of the design philosophy behind it. Really minimal
  • Adélie Linux - Another smallish project, but not rolling release. It's aiming to be fully POSIX-compliant as well which is pretty cool. It shares some ideas with Alpine w.r.t. the package manager, but I much prefer the 'standard' implementation that Adélie goes with (though Alpine is worth checking out as well, of course)

Software

There's also various neat pieces of software I've come across, which I'll be going over.

NB that my definition of software is very lax; take it to just mean some collection of code (no matter how large or small) which does something together. Again, at some point I'll talk about design **todo**.

  • lf - a TUI-based file manager. If you've heard of ranger, it's very similar to that, but has some other features and I prefer the modularity of lf more.
  • dmenu - a menu tool. Works very similar to the 'Finder' tool on MacOS; I use it to launch *.desktop files. Worth looking at rofi or maybe even a terminal based alternative instead.
  • cmus - A music player. Not actively maintained as of early 2019, but much simpler (and nicer) to use than mpd or other alternatives, imo.
